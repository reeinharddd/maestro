# Stale Test Files Analysis

Three pre-existing `_test.go` files no longer compile. This document analyzes each and recommends fix or delete.

---

## 1. `internal/db/db_test.go` — DELETE

### Current Errors

```
undefined: db.CreateDB
```

8 instances of `db.CreateDB(":memory:")` — this function was removed when the API changed to `db.Open(path)`.

### Additional Issues (hidden by first error)

- References `db.Provider`, `db.Model` — these types moved to `pkg/models.Provider`, `pkg/models.Model`
- References `db.ProviderID(...)`, `db.Search(...)` — these filter functions don't exist in the current `models.go`
- References `db.StatusActive`, `db.StatusInactive` — these are functions (`StatusActive()`, not a type)
- Uses `db.UpsertProvider(db, provider)` — current signature is `(d *DB) UpsertProvider(p *models.Provider) error`
- Uses `testify/assert` — not a direct dependency (only indirect)

### Verdict: DELETE

The test was written against an older API surface that was completely refactored. Too many mismatches to salvage. The test patterns are good and should be rewritten from scratch using the current API:

```go
// Replacement pattern:
func setupTestDB(t *testing.T) *db.DB {
    t.Helper()
    d, err := db.Open(":memory:")
    if err != nil {
        t.Fatalf("open test db: %v", err)
    }
    t.Cleanup(func() { d.Close() })
    return d
}
```

## 2. `internal/heal/heal_test.go` — DELETE

### Current Errors

```
unknown field SessionID in struct literal of type heal.HealReport
undefined: heal.StatusSuccess
unknown field Status in struct literal of type heal.HealReport
unknown field Message in struct literal of type heal.HealReport
```

### Analysis

The `HealReport` struct was completely redesigned. Old version:

```go
// Old (as expected by the test)
type HealReport struct {
    SessionID string
    Status    string  // heal.StatusSuccess
    Message   string
}
```

Current version:

```go
// Current (actual)
type HealReport struct {
    IssuesFound int
    IssuesFixed int
    Issues      []HealIssue
}
```

The test creates a report with session tracking fields that no longer exist. The `HealReport` is now generated by `Service.Run()` and tracks issue counts, not session status.

### Verdict: DELETE

The `HealReport` struct was fundamentally redesigned. The test should be rewritten to test `Service.Run()` with a real database instance:

```go
// Replacement pattern:
func TestHealRun(t *testing.T) {
    d := setupTestDB(t)
    s := heal.New(d)
    report, err := s.Run(context.Background())
    // assert no error, check IssuesFound/IssuesFixed
}
```

## 3. `internal/routing/router_test.go` — DELETE

### Current Errors

```
undefined: routing.TaskType
undefined: routing.TaskRequirements
undefined: routing.TaskTypeCode
undefined: routing.GetTaskRequirements
```

### Analysis

The old routing package used an enum-based approach:

```go
// Old (as expected by the test)
type TaskType int
const (
    TaskTypeCode  TaskType = iota
    TaskTypeChat
    TaskTypeImage
    ...
)
type TaskRequirements struct {
    Code  bool
    Chat  bool
    Image bool
    ...
}
func GetTaskRequirements(t TaskType) TaskRequirements { ... }
```

The current routing package uses string-based task keys and the `Service.SelectBestModel()` method:

```go
// Current (actual)
type Service struct { db *db.DB }
func (s *Service) SelectBestModel(taskType string, budget models.BudgetConfig) (*models.RoutingRule, error)
```

### Verdict: DELETE

The routing package was redesigned from an enum-based static system to a service-based dynamic system. Replace with tests for `Service.SelectBestModel()`:

```go
// Replacement pattern:
func TestSelectBestModel(t *testing.T) {
    d := setupTestDB(t)
    s := routing.New(d)
    rule, err := s.SelectBestModel("coding_fast", models.BudgetConfig{PreferredTier: "free_only"})
    // assert NoError, rule.TaskKey == "coding_fast", rule.CurrentModelID != ""
}
```

## Summary

| File | Verdict | Reason |
|------|---------|--------|
| `internal/db/db_test.go` | DELETE | API completely refactored (CreateDB → Open, types moved to pkg/models) |
| `internal/heal/heal_test.go` | DELETE | HealReport struct completely redesigned |
| `internal/routing/router_test.go` | DELETE | Routing changed from enum-based to service-based |

After deleting these 3 files, `go vet ./...` and `go test -race ./...` will pass cleanly. The test patterns they used are documented in `DEVELOPMENT.md` as reference for future replacement tests.

### If You Prefer to Fix Instead

Fixing is not recommended because the underlying APIs changed so fundamentally that fixing would effectively be a full rewrite. The new tests should follow the patterns in [DEVELOPMENT.md](./DEVELOPMENT.md#5-test-methodology) (table-driven, external package, SQLite in-memory, `t.Helper()`).
